# PROJECT BRIEF

## Контекст

CLR Plugin нужен для удобного обмена дизайн-токенами между внешним JSON-описанием и Figma Variables, а также для автоматического создания понятной документации прямо в Figma-файле.

## Цели проекта

- Сделать надёжный импорт токенов из JSON в Figma Variables.
- Сделать обратимый экспорт Variables в JSON без потери структуры.
- Автоматизировать генерацию токен-документации в виде фреймов.

## Целевая аудитория

- Дизайн-систем команда.
- Продуктовые дизайнеры.
- Фронтенд-разработчики, которые используют JSON-токены в коде.

## Scope MVP

1. Импорт:
   - Чтение JSON в согласованной структуре.
   - Создание/обновление коллекций, режимов и переменных в Figma.
   - Поддержка alias/reference токенов.
   - Поддержка градиент-токенов через локальные Paint Styles (из-за ограничений Variables API).
   - Полная sync-очистка по source of truth: удаление переменных и коллекций, отсутствующих во входном JSON.
   - Нормализация имен градиент-стилей по продуктовой конвенции (`<Product> Gradients/...`).
2. Экспорт:
   - Чтение Variables из файла.
   - Чтение локальных Paint Styles для `$type: "gradient"`.
   - Сериализация в тот же JSON-формат с unified-коллекциями (`Common`, `Core`, `Product`, `External`).
3. Документация:
   - Генерация фреймов по коллекциям (попродуктно) по фиксированному шаблону.
   - Базовые превью значений (цвет, текст, размеры, радиусы).

## Out of Scope (на старте)

- Интеграция с удалённым API/бэкендом.
- Работа с git прямо из плагина.
- Сложные темы прав доступа и multi-workspace синхронизация.
- Полная совместимость со всеми фичами Tokens Studio.

## Критерии успешного MVP

- Один и тот же JSON после `import -> export` сохраняет структуру и ключевые значения.
- Документация-фреймы создаются автоматически и читаемы без ручной доработки.
- Операции работают на реальных файлах дизайн-системы без критических падений.
