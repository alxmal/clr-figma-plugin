# ROADMAP

## Phase 0 - Discovery and Specs

- Утвердить JSON-схему и правила маппинга.
- Зафиксировать шаблон документации-фреймов.
- Подтвердить стэк и структуру проекта.

**Выход:** подписанные документы в `docs/`.

## Phase 1 - Project Bootstrap

- Инициализация TypeScript + React + Vite.
- Подключение Figma plugin typings.
- Настройка ESLint/Prettier/Vitest.
- Базовый канал сообщений между UI и main.

**Выход:** запускаемый каркас плагина.

## Phase 2 - Import Engine

- Zod-валидация JSON.
- Преобразование в внутреннюю модель токенов.
- Создание/обновление коллекций, режимов, переменных.
- Отчёт об ошибках/предупреждениях в UI.

**Выход:** рабочий импорт для поддерживаемых типов токенов.

## Phase 3 - Export Engine

- Чтение Variables из Figma.
- Нормализация в общий формат.
- Экспорт JSON с сохранением структуры.

**Выход:** обратимый цикл `import -> export`.

## Phase 4 - Documentation Generator

- Генерация корневого набора фреймов по шаблону.
- Рендер карточек токенов по категориям.
- Базовая оптимизация скорости генерации.

**Выход:** автодокументация внутри Figma.

## Phase 5 - Stabilization

- Тесты на реальных наборах токенов.
- UX-полировка UI.
- Подготовка к внутреннему релизу.

**Выход:** кандидат на production-использование.

## Phase 6 - Advanced Authoring UX

- Добавить подсветку синтаксиса JSON в существующем поле ввода конфигурации в UI плагина.
- Добавить node-based конструктор конфигов (React Flow) с визуальным редактированием токен-графа.
- Поддержать двустороннюю синхронизацию: `JSON <-> node graph` с импортом/экспортом без потери структуры.

**Выход:** визуальный и текстовый режимы редактирования конфигов в одном плагине.

## Phase 7 - Developer Export Pipeline

- Добавить отдельный transformer для выгрузки design tokens в dev-friendly артефакты.
- Поддержать как минимум:
  - `tokens.css` (CSS custom properties с `var(...)`),
  - `tokens.resolved.json` (alias уже раскрыты),
  - сериализацию gradient-токенов в формат для frontend (объект и/или CSS gradient string).
- Добавить CLI-команду (`npm run export:dev`) и документацию по интеграции в CI.

**Выход:** стабильный pipeline для доставки токенов разработчикам без ручной постобработки.

## Current Status Notes

- Import/export для variables и gradients реализован.
- Поддержан multi-product подход через unified-коллекции `Common` / `Core` / `Product` / `External`.
- Добавлена утилита миграции legacy JSON в multi-product структуру.
- Документация генерируется по коллекциям и учитывает gradient styles.
- Feature request зафиксирован: отдельная dev-выгрузка (CSS/JSON) как следующий этап.
