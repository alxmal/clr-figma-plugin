# DOCUMENTATION FRAMES SPEC

Спецификация генерации документации токенов внутри Figma-файла.

## Цель

Автоматически создавать читаемые фреймы с описанием токенов и примерами значений, чтобы команда могла быстро проверить и использовать токены.

## Общий шаблон

Генерируется корневой фрейм:

- `CLR Tokens Documentation`

Внутри:

1. `00 Cover` - заголовок, версия формата, дата генерации.
2. `01 Colors` - карточки color токенов.
3. `02 Typography` - текстовые стили/параметры (если есть).
4. `03 Spacing` - шкала отступов.
5. `04 Radius` - радиусы.
6. `05 Misc` - прочие типы токенов.

## Карточка токена (минимум)

Каждая карточка включает:

- Token name (полный путь, например `color.brand.primary`)
- Тип (`color`, `number`, `string`...)
- Значение по режимам (`Light`, `Dark`, ...)
- Description (если есть)

Дополнительно:

- Для `color` - swatch/заливка.
- Для `number` - визуализация значения в px/rem по выбранной политике.

## Layout правила

- Auto Layout для всех контейнеров.
- Консистентные отступы (например 16/24).
- Ограничение ширины карточки для читаемости.
- Сортировка токенов по имени (алфавитно).

## Поведение генератора

- Команда `Generate docs`:
  - ищет существующий корневой фрейм по имени,
  - если найден, обновляет существующую структуру (без полного пересоздания),
  - если не найден, создаёт корневой фрейм и вложенные секции,
  - пишет метку времени последней генерации.

## MVP-ограничения

- Без кастомного редактирования шаблона через UI.
- Без публикации в сторонние форматы (PDF/HTML).

## DoD для документации

- После импорта токенов можно одним кликом сгенерировать полный набор фреймов.
- Фреймы создаются без ошибок на файлах с 200+ токенами.
- Результат читаем дизайнером без необходимости смотреть JSON.
